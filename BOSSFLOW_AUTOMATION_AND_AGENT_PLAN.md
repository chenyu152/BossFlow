# BossFlow 自动化与 Agent 能力演进方案

## 1. 目标与边界

BossFlow 的自动化能力分为两层：

1. **确定性自动化**：按计划触发岗位采集，保证排队、互斥、恢复、日志和失败可见。
2. **智能体自动化**：由 Agent 根据用户目标组合采集、筛选、精评、简历材料、面试准备和故事库导入等能力。

两者共享同一套持久化任务队列。Agent 不绕过任务队列直接操作爬虫，也不直接取得桌面应用的运行令牌。

## 2. 定时采集设计

### 2.1 调度模型

每条采集计划包含：

- 求职方向（项目）
- 是否启用
- 执行频率：每天、工作日、每周指定日期
- 本地执行时间
- 错过执行策略：恢复后补跑一次、直接跳过
- 最大允许延迟时间
- 下次执行时间、最近执行结果

计划到期后先写入持久化队列，再由单个采集 Worker 串行执行。排序规则为：

1. 原计划执行时间
2. 入队时间
3. 稳定 ID 顺序

相同时间的多个求职方向不会并发访问招聘平台。任务 1 未完成时，任务 2 保持 `queued` 状态并显示“等待前序任务完成”。手动任务运行期间，定时任务同样等待。

### 2.2 防止任务积压

- 同一计划最多存在一个 `queued` 或 `running` 实例。
- 应用休眠或关闭后恢复时，根据错过策略补跑一次或记录为 `missed`。
- 不为每一个错过的周期重复补建任务。
- 验证码、登录失效和安全校验应进入需要用户处理的失败状态，不持续高频重试。
- 创建或启用计划前检查对应求职方向的持久化 BOSS Cookie；没有核心登录 Cookie 时阻止启用并引导用户先登录。
- 每次成功验证登录后记录时间，同时读取 Chromium Cookie 自身的到期元数据。登录超过 3 天提示建议刷新，但不宣称固定有效期；BOSS 服务端可能提前使会话失效。
- 每个排队任务真正启动前再次检查 Cookie。不可用时将该次运行标记为失败并给出“登录 / 保存 Cookie”的可执行提示，不启动浏览器采集。

### 2.3 运行状态

- `queued`：已入队，等待前序任务
- `running`：正在采集
- `succeeded`：成功完成
- `failed`：执行失败
- `missed`：超过补跑窗口或配置为跳过
- `interrupted`：应用异常退出时任务仍在运行
- `cancelled`：用户取消

### 2.4 设置页

“自动化与后台运行”区域提供：

- 开机/登录系统后启动 BossFlow
- 启动时最小化到系统托盘
- 关闭主窗口后保持后台运行
- 定时任务新建、编辑、启停、删除和立即运行
- 下次执行时间、当前队列状态和最近运行记录

主界面不暴露 Cron 表达式，优先使用每天、工作日和每周日期选择。

## 3. 桌面后台运行

开机启动由 Electron 主进程管理；后台调度依赖桌面应用和后端 sidecar 持续存活，因此需要托盘模式：

- 关闭窗口时根据设置隐藏到托盘，而不是退出进程。
- 托盘菜单提供“打开主窗口”和“退出 BossFlow”。
- 从开机启动进入时可携带隐藏参数，不打断用户当前工作。
- 用户明确退出时，停止当前任务并关闭 sidecar。

当前方案是“用户登录系统后运行”，不安装 Windows 系统服务。爬虫依赖用户浏览器配置、登录态和交互式安全校验，不适合运行在无人登录的系统服务会话中。

## 4. 并发与平台风险

招聘平台采集默认全局串行。多账号并行不会消除平台协议、账号安全、设备指纹、Cookie 隔离和验证码风险，因此不作为产品功能提供。

本地后处理可以采用有限并行，例如清洗、评分、LLM 精评和材料生成，但必须避免多个任务同时修改同一岗位或同一材料文件。

## 5. Agent 与 Skills 演进

### 5.1 分层

| 层级 | 作用 | 示例 |
| --- | --- | --- |
| Tool / MCP | 稳定、可审计的原子能力 | 查询岗位、加入候选、生成材料 |
| Skill | 面向用户目标的工作流说明 | 处理今日新增岗位、准备某岗位申请 |
| Workflow | 可恢复的确定性步骤 | 采集 → 初筛 → 精评 → 候选 → 材料 |
| Agent | 理解目标并选择 Skill / Workflow | “帮我处理今天新增的岗位” |

### 5.2 首批 Skill

- `discover-jobs`：为指定求职方向采集岗位
- `triage-new-jobs`：筛选新增岗位并形成候选建议
- `prepare-application`：完成精评、简历材料和面试准备
- `import-story-bank`：从用户授权的项目目录提取故事草稿
- `daily-job-review`：汇总当天新增岗位和待处理事项

### 5.3 外部 Agent 接入

BossFlow 核心能力通过本地 MCP/CLI 暴露，Claude Code、Trae 等仅加载各自的 Skill 适配层。外部 Agent 不直接访问内部数据库或 Electron 运行令牌。

故事库导入遵守：

- 用户显式选择允许读取的目录
- 默认只读，不修改用户原项目
- 保留来源文件与引用片段
- 推断内容与原文事实分开标记
- 先进入草稿箱，用户确认后写入正式故事库

## 6. 实施顺序

1. 持久化任务队列、运行历史和单 Worker 互斥
2. 定时计划 API 与设置页
3. Electron 开机启动、隐藏启动和托盘后台运行
4. 采集完成后的确定性工作流
5. 本地 MCP/CLI 与 Agent Skills

第 1～3 项已经完成。Agent 第一阶段也已进入实现：FastAPI 挂载本地 Streamable HTTP MCP，提供 stdio 薄桥、确认式写工具、审计日志，以及岗位筛选、申请准备和故事库导入三套 Skills。后续继续补充内置 Agent UI、更多客户端适配和长任务进度通知。
