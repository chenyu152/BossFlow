# Figma Design Prompt: BossSpider Web Console

请设计一个用于 BossSpider 的 Web 控制台。此阶段只需要定义信息架构、页面和组件，不需要决定具体视觉风格。

## 产品目标

BossSpider 是一个本地运行的 BOSS 直聘岗位采集和岗位数据管理工具。Web 控制台用于替代旧 PyQt/QML 桌面界面，让用户能配置采集范围、启动爬虫、查看实时日志、浏览 SQLite 岗位结果，并导出数据。

## 页面结构

需要 5 个主页面，可用左侧导航或顶部标签切换：

1. 仪表盘
2. 采集范围
3. 清洗规则
4. 数据浏览
5. 运行日志

页面应共享一个全局项目选择器，用于选择当前岗位项目，例如 `agent`、`cehua`、`产品经理`。项目切换后要刷新配置、统计数据和岗位列表。

## 全局组件

- 项目选择器：展示可用项目名称；切换后加载对应 `config.json`、`jobs_data.db`、`crawl_partial.json`。
- 运行状态指示：显示 `ready`、`crawling`、`login`、`processing-partial`、`stopping`、`failed`。
- 主操作区：在任何页面都能看到或快速进入“启动采集/终止采集”和“查看日志”的入口。
- Toast/错误提示：用于配置格式错误、任务冲突、导出失败、接口错误等。

## 1. 仪表盘

展示当前项目的概览和核心操作。

组件：
- 指标卡片：本地岗位总数、关键词数量、城市数量、当前数据库路径。
- 采集策略选择：三段式控件，选项为“标准模式”“贪婪模式”“滚动模式”。
- 运行开关：
  - 快速模式：从关键词中随机抽取部分关键词。
  - 无头模式：后台运行浏览器。
  - 自动写入 SQLite：采集后清洗并写入本地数据库。
- 主按钮：
  - 未运行时显示“启动采集”。
  - 运行中显示“终止采集”。
- 次级按钮：
  - “登录 / 保存 Cookie”
  - “从中断 JSON 恢复”
- 最近日志预览：显示最新几行日志，并提供跳转到运行日志页。

## 2. 采集范围

用于编辑采集输入和抓取上限。

组件：
- 关键词多行编辑器：每行一个关键词。
- 城市快捷选择区：常用城市按钮，北京、上海、广州、深圳、杭州、成都、武汉、南京、西安、长沙、重庆、苏州。
- 城市手动编辑器：每行格式为 `城市名=城市代码`。
- 抓取上限表单：
  - 标准翻页页数
  - 滚动目标条数
  - 最大滚动次数
- 保存配置按钮。
- 加载/刷新配置按钮。

## 3. 清洗规则

用于编辑采集后的岗位过滤与分类规则。

组件：
- 分类匹配规则 JSON 编辑器：键为分类名，值为关键词数组。
- 相关性兜底关键词编辑器：未命中分类时，标题包含这些词仍保留。
- 黑名单关键词编辑器：标题包含这些词时过滤。
- 最低均薪输入：单位 K/月。
- JSON 校验反馈：显示分类规则是否为合法 JSON 对象。
- 保存配置按钮。

## 4. 数据浏览

用于检索和预览 SQLite 中的岗位。

组件：
- 搜索框：搜索岗位名、公司名、岗位描述。
- 刷新按钮。
- 导出 Excel 按钮。
- 岗位表格：
  - ID
  - 岗位名称
  - 公司名称
  - 城市
  - 薪资
  - 平均薪资 K
  - 经验
  - 学历
  - 分类
  - 最后活跃时间
- 右侧或抽屉式岗位详情预览：
  - 岗位名称
  - 公司、城市
  - 薪资、经验、学历
  - 平均薪资
  - 分类标签
  - 岗位描述
  - 原始链接
- 分页或“加载更多”，因为岗位数可能超过 1000。

## 5. 运行日志

用于实时观察后端任务。

组件：
- 实时日志输出区域，支持等宽字体。
- 自动滚动到底部开关。
- 清空前端显示按钮。
- 任务状态栏：运行中、就绪、失败、正在终止。
- 当前任务摘要：任务类型、项目名、Chrome profile 路径、debug port、开始时间。

## 主要交互流程

1. 用户进入仪表盘，选择项目。
2. 系统加载配置、统计数据和岗位列表。
3. 用户在采集范围页编辑关键词、城市和上限。
4. 用户在清洗规则页编辑分类规则、相关性关键词、黑名单和最低薪资。
5. 用户回到仪表盘选择采集策略和运行开关。
6. 用户点击启动采集，自动跳到运行日志页。
7. 运行期间主按钮变成终止采集，日志持续追加。
8. 任务结束后刷新岗位统计和数据浏览表格。
9. 用户在数据浏览页搜索、查看详情、导出 Excel。

## 后端接口参考

- `GET /api/projects`
- `GET /api/config?project=agent`
- `POST /api/config`
- `GET /api/stats?project=agent`
- `GET /api/jobs?project=agent&q=&limit=500&offset=0`
- `GET /api/jobs/export?project=agent&q=`
- `POST /api/tasks/crawl`
- `POST /api/tasks/login`
- `POST /api/tasks/process-partial`
- `POST /api/tasks/stop`
- `GET /api/tasks/status`
- `GET /api/logs?since=0`
- `GET /api/logs/stream`

## 约束

- 不需要设计营销页或介绍页，首屏就是可操作控制台。
- 页面应服务于高频操作和长时间运行监控。
- 不要把说明文字做得很重，字段标签和错误提示清晰即可。
- 数据表需要适合快速扫描、筛选和查看详情。
